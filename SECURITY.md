# Security policy

## Reporting a vulnerability

Please do not report security vulnerabilities in public GitHub issues. Instead, use GitHub's private vulnerability-reporting flow for this repository. If that option is unavailable, contact the repository owner through their GitHub profile with a concise report and a safe reproduction path.

Include the affected area, impact, steps to reproduce, and any suggested mitigation. We will acknowledge a valid report as soon as practical and coordinate disclosure after a fix is available.

## Scope

High-priority areas include authentication and ambassador authorization, artifact validation and redaction, project ownership checks, private R2 media access, Cloudflare/D1 configuration, and accidental exposure of credentials or personal conversation data.
